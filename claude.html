What is Claude Code?

Claude Code is an AI assistant that works directly with your files and folders. It can read documents, help you write and edit, organize research, and much more.

  • Read and understand your documents
  • Help draft, edit, and refine writing
  • Organize and summarize research materials
  • Create outlines, syllabi, and course materials
  • Analyze data and generate reports
  • Answer questions about your files

Opening Claude Code (The Easy Way)

Step 1: Navigate to Your Folder in File Explorer

  1. Open File Explorer (press Win + E)
  2. Navigate to the folder containing your project or documents
  3. For example: Documents ‚Üí Research ‚Üí My Paper

Step 2: Right-Click ‚Üí Open Git Bash Here

  1. Once you're in your folder, right-click on empty space
  2. Click "Open Git Bash here"
  3. A terminal window will open, already in that folder
Tip: Hold Shift while right-clicking for additional options.

Step 3: Type claude and Press Enter

claude

That's it! Claude will start and you can begin working.

üöÄ Supercharge Your Raspberry Pi

Turn your Pi-hole into an always-on AI gateway. Access Claude from anywhere, get notifications, and have an assistant that's always running.

For Claude Code: You can read this page and follow the instructions step by step. Run each command block in order and verify the expected output before proceeding.

Prerequisites Checklist

  • Raspberry Pi 5 with Pi-hole installed and running
  • Pi connected to your network (ethernet recommended)
  • SSH access to your Pi working
  • Your Pi's IP address (check Fing app or router)

Part 1: Connect to Your Pi

Step 1.1: SSH into your Pi

From Git Bash on your PC, connect to the Pi:

ssh pihole@YOUR_PI_IP

Replace YOUR_PI_IP with your Pi's actual IP address (e.g., 192.168.1.55)

Default password: raspberry

Expected output: You should see a command prompt like pihole@pihole:~$

Part 2: Install Tailscale (Remote Access)

Tailscale lets you access your Pi from anywhere ‚Äî home, work, phone, anywhere.

Step 2.1: Install Tailscale

Run this command on your Pi:

curl -fsSL https://tailscale.com/install.sh | sh

Expected output: Installation messages ending with "Installation complete"

Step 2.2: Start Tailscale as Exit Node

This lets you route all traffic through your Pi (for ad-blocking anywhere):

sudo tailscale up --advertise-exit-node

Expected output: A URL to authenticate. Open it in your browser.

Step 2.3: Authenticate

  1. Copy the URL from the terminal
  2. Open it in your browser
  3. Sign in with Google (or create a Tailscale account)
  4. Authorize the device

Expected result: Terminal shows "Success" and your Pi is connected.

Step 2.4: Approve Exit Node

  1. Go to Tailscale Admin Console
  2. Find your Pi in the machines list
  3. Click the ‚ãÆ menu ‚Üí Edit route settings
  4. Enable "Use as exit node"
  5. Save

Step 2.5: Get Your Pi's Tailscale IP

tailscale ip -4

Expected output: An IP like 100.x.x.x ‚Äî save this!

Success! You can now access your Pi from anywhere using the Tailscale IP. Install Tailscale on your phone and laptop to connect from anywhere.

Part 3: Install Node.js

OpenClaw requires Node.js. Let's install it.

Step 3.1: Add NodeSource Repository

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo bash -

Expected output: Repository added successfully

Step 3.2: Install Node.js

sudo apt install -y nodejs

Expected output: Installation completes without errors

Step 3.3: Verify Installation

node --version
npm --version

Expected output: Version numbers like v22.x.x and 10.x.x

Part 4: Install OpenClaw

OpenClaw is an AI gateway that gives you Claude access via Telegram, Discord, or web chat.

Step 4.1: Install OpenClaw

sudo npm install -g openclaw

Expected output: Installation completes, shows openclaw version

Step 4.2: Initialize OpenClaw

openclaw init

This will ask you for:

  • Anthropic API key ‚Äî get one from console.anthropic.com
  • Default model ‚Äî recommend claude-sonnet-4 for balance of speed/quality

Step 4.3: Configure Network Binding

To access OpenClaw from other devices, edit the config:

nano ~/.openclaw/openclaw.yaml

Find the gateway section and set:

gateway:
  port: 18789
  bind: tailnet  # or 0.0.0.0 for all interfaces

Save with Ctrl+O, exit with Ctrl+X

Step 4.4: Start OpenClaw

openclaw gateway start

Expected output: Gateway starts, shows URL

Step 4.5: Test It

From your PC or phone (with Tailscale connected), open a browser:

http://YOUR_TAILSCALE_IP:18789

Expected result: OpenClaw web chat interface loads!

Part 5: Run OpenClaw on Startup

Make OpenClaw start automatically when your Pi boots.

Step 5.1: Create systemd Service

sudo nano /etc/systemd/system/openclaw.service

Paste this content:

[Unit]
Description=OpenClaw AI Gateway
After=network.target

[Service]
Type=simple
User=pihole
ExecStart=/usr/bin/openclaw gateway start --foreground
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target

Save and exit.

Step 5.2: Enable and Start

sudo systemctl daemon-reload
sudo systemctl enable openclaw
sudo systemctl start openclaw

Step 5.3: Check Status

sudo systemctl status openclaw

Expected output: Shows "active (running)"

Part 6: Optional ‚Äî Connect Telegram

Chat with Claude from your phone via Telegram.

Step 6.1: Create a Telegram Bot

  1. Open Telegram and message @BotFather
  2. Send /newbot
  3. Choose a name (e.g., "Mason's Claude")
  4. Choose a username (must end in "bot", e.g., "masons_claude_bot")
  5. Copy the API token BotFather gives you

Step 6.2: Add to OpenClaw Config

nano ~/.openclaw/openclaw.yaml

Add under channels:

channels:
  telegram:
    token: "YOUR_BOT_TOKEN_HERE"
    allowlist:
      - "your_telegram_username"

Step 6.3: Restart OpenClaw

sudo systemctl restart openclaw

Now message your bot on Telegram ‚Äî Claude will respond!

Troubleshooting

Can't connect to Pi via Tailscale

# On the Pi, check Tailscale status
tailscale status

# Make sure it's running
sudo systemctl status tailscaled

OpenClaw won't start

# Check logs
sudo journalctl -u openclaw -f

# Check config syntax
openclaw config validate

Telegram bot not responding

  • Make sure your username is in the allowlist
  • Check the bot token is correct
  • Restart OpenClaw after config changes
