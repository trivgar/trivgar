Pi-hole + Unbound Setup Guide

Network-wide ad blocking and private DNS resolution

What This System Does

Pi-hole

Pi-hole is a network-wide DNS filter. Every device on your network asks it where websites and services live on the internet.

  • Blocks ads, trackers, and telemetry before they load
  • Works for phones, TVs, tablets, and computers
  • No browser extensions required
  • Reduces tracking and network noise

Unbound

Unbound is a local, private DNS resolver that runs behind Pi-hole. Instead of asking Google or your ISP for DNS answers, it queries the internet’s root DNS servers directly.

  • No third-party DNS provider
  • No external logging of your DNS requests
  • Local caching for faster repeat lookups

Together

Device → Pi-hole → Unbound → Internet (root DNS)

Pi-hole blocks what should never load. Unbound privately resolves everything else.

Hardware Setup

  1. Connect the Raspberry Pi to your router using an Ethernet cable.
  2. Plug in the USB-C power cable.
  3. Wait about 60 seconds for the Pi to boot.

A solid green light means it’s ready.

Find the Pi’s IP Address

  1. Log into your router’s admin page.
  2. Find the connected devices list.
  3. Look for a device named pihole or raspberrypi.
  4. Write down the IP address (example: 192.168.1.55).

Reserve the IP Address (Required)

Prevent the Pi-hole IP from changing.

  1. In your router, find DHCP Reservation / Static Lease.
  2. Select the Pi-hole device.
  3. Reserve the IP address.

Set the Router DNS

  1. Find DNS settings in your router.
  2. Set Primary DNS to the Pi-hole IP.
  3. Leave Secondary DNS blank or set it to the same IP.
If you leave 8.8.8.8, 1.1.1.1, or anything else as secondary DNS, ads and trackers will bypass Pi-hole.

Save settings and allow the router to reboot if needed.

Verify Pi-hole Is Working

  1. Open a browser on your phone or computer.
  2. Go to: http://<PIHOLE_IP>/admin
  3. Log in using the admin password provided with the device.
If the dashboard loads and shows queries being blocked, everything is working.

SSH Access (Optional)

Username: pihole
Password: raspberry

SSH is only needed for maintenance or advanced configuration. Normal operation does not require logging in.
