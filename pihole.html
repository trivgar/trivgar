Pi-hole + Unbound Setup Guide

Network-wide ad blocking with private DNS resolution

What This Does

Pi-hole

Pi-hole is a network-wide DNS sinkhole. Every device on your network asks it where websites and services live on the internet.

  • Blocks ads, trackers, and telemetry before they load
  • Works for phones, TVs, tablets, computers — everything on your network
  • No browser extensions required
  • Reduces bandwidth and speeds up page loads

Unbound

Unbound is a local recursive DNS resolver that runs behind Pi-hole. Instead of asking Google (8.8.8.8) or Cloudflare (1.1.1.1) for DNS answers, it queries the internet's root DNS servers directly.

  • No third-party DNS provider sees your queries
  • No external logging of your browsing
  • Local caching for faster repeat lookups
  • DNSSEC validation for security

The Flow

Device → Pi-hole (blocks ads) → Unbound (resolves privately) → Root DNS

Requirements

  • Raspberry Pi (3B+ or newer recommended) or any Linux machine
  • Ethernet connection to your router (WiFi works but ethernet is more reliable)
  • Access to your router's admin panel

Install Pi-hole

SSH into your Pi or Linux machine and run:

curl -sSL https://install.pi-hole.net | bash
Follow the installer prompts. The defaults are fine for most setups. Write down the admin password it generates at the end.

Install Unbound

sudo apt update
sudo apt install unbound -y

Download the root hints file:

sudo wget -O /var/lib/unbound/root.hints https://www.internic.net/domain/named.root

Create the Pi-hole config for Unbound:

sudo nano /etc/unbound/unbound.conf.d/pi-hole.conf

Paste this configuration:

server:
    verbosity: 0
    interface: 127.0.0.1
    port: 5335
    do-ip4: yes
    do-udp: yes
    do-tcp: yes
    do-ip6: no
    prefer-ip6: no
    
    # Security
    harden-glue: yes
    harden-dnssec-stripped: yes
    use-caps-for-id: no
    edns-buffer-size: 1232
    
    # Privacy
    qname-minimisation: yes
    
    # Performance
    num-threads: 1
    msg-cache-slabs: 2
    rrset-cache-slabs: 2
    infra-cache-slabs: 2
    key-cache-slabs: 2
    rrset-cache-size: 100m
    msg-cache-size: 50m
    
    private-address: 192.168.0.0/16
    private-address: 169.254.0.0/16
    private-address: 172.16.0.0/12
    private-address: 10.0.0.0/8
    private-address: fd00::/8
    private-address: fe80::/10

Restart Unbound:

sudo systemctl restart unbound

Test it:

dig google.com @127.0.0.1 -p 5335
If you get a response with an IP address, Unbound is working.

Configure Pi-hole to Use Unbound

  1. Open Pi-hole admin: http://<PI_IP>/admin
  2. Go to Settings → DNS
  3. Uncheck all upstream DNS servers (Google, Cloudflare, etc.)
  4. Under "Custom 1 (IPv4)", enter: 127.0.0.1#5335
  5. Save

Set Static IP on the Pi

The Pi needs a static IP so your router always knows where to send DNS queries.

Edit dhcpcd.conf:

sudo nano /etc/dhcpcd.conf

Add at the bottom (adjust for your network):

interface eth0
static ip_address=192.168.1.10/24
static routers=192.168.1.1
static domain_name_servers=127.0.0.1

Reboot:

sudo reboot

Configure Your Router

  1. Log into your router's admin panel
  2. Find DNS settings (usually under DHCP or WAN settings)
  3. Set Primary DNS to your Pi's IP address
  4. Leave Secondary DNS blank or set it to the same IP
  5. Save and reboot the router
Important: If you add a secondary DNS like 8.8.8.8 or 1.1.1.1, devices may bypass Pi-hole entirely and you'll still see ads.

Verify It's Working

  1. Visit http://<PI_IP>/admin
  2. You should see queries being logged and blocked
  3. Visit ads-blocker.com/testing to test ad blocking
If the dashboard shows queries and blocks, you're done! All devices on your network are now protected.

Optional: Add More Blocklists

Pi-hole's default blocklist is good, but you can add more:

  1. Go to Adlists in Pi-hole admin
  2. Add URLs from firebog.net
  3. Run Tools → Update Gravity

Troubleshooting

No internet after setup?

  • Check Pi-hole is running: pihole status
  • Check Unbound is running: sudo systemctl status unbound
  • Temporarily set router DNS back to 8.8.8.8 to restore internet while debugging

Ads still showing?

  • Make sure secondary DNS is blank in router settings
  • Flush DNS cache on your device
  • Some apps use hardcoded DNS (YouTube, etc.) — these bypass Pi-hole

Website broken?

  • Check Query Log in Pi-hole admin to see what's being blocked
  • Whitelist the domain if needed
